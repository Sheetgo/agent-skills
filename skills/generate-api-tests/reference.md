# Generate API Tests — Reference

Heavy reference for templates, formats, and expression syntax. Read this file when generating tests or running init.

## Expression Syntax (go-runner / expr-lang)

| Pattern | Syntax | Example |
|---------|--------|---------|
| Environment var | `${{ env.VAR }}$` | `${{ env.MAIN_FILE }}$` |
| Simple env substitution | `${VAR}` | `${MAIN_FILE}` |
| Variable from previous test | `${{ vars.id }}$` | `${{ vars.connection_id }}$` |
| Response status | `${{ response.Status }}$` | Capitalized (Go struct field) |
| JSON response field | `${{ json.field }}$` | `${{ json.values.id }}$` |
| Fixtures data | `${{ fixtures.key }}$` | `${{ fixtures.users.admin.email }}$` |
| Loop data | `${{ loopData }}$` | Current iteration value |
| String concatenation | `+` operator | `${{ vars.prefix + "-suffix" }}$` |
| Array access | `[index]` | `${{ json.items[0].name }}$` |
| Nil check | `!= nil` | `${{ json.data != nil }}$` |
| Empty string check | `!= ""` | `${{ json.id != "" }}$` |
| Array length | `len()` | `${{ len(json.items) > 0 }}$` |
| Type conversion | `string()`, `int()` | `${{ string(loopData) }}$` |

**Key differences from JavaScript:** `response.Status` (not `response.status()`), `env.VAR` (not `process.env.VAR`), `!= nil` (not `!== undefined`), `len(arr)` (not `arr.length`).

**Fixtures:** Loaded from `resorce.json` in the spec file's directory (note: filename preserves a legacy typo). Returns a JSON map accessible via `${{ fixtures.path.to.value }}$`. See CLAUDE.md Fixtures System section for details.

## Framework Detection Signals

| Framework | Detection Signals |
|-----------|-------------------|
| Flask | `from flask import`, `@app.route`, `Blueprint`, `Flask(__name__)` |
| FastAPI | `from fastapi import`, `@app.get`, `@router.post`, `APIRouter` |
| Django | `urls.py` with `path()`, `views.py`, `from django.http` |
| Express | `express()`, `app.get(`, `app.post(`, `router.get(` |
| NestJS | `@Controller`, `@Get()`, `@Post()`, `@Injectable()` |
| Go (stdlib) | `http.HandleFunc`, `http.ListenAndServe` |
| Go (Gin) | `gin.Default()`, `gin.New()`, `r.GET(`, `r.POST(` |
| Go (Echo) | `echo.New()`, `e.GET(`, `e.POST(` |

**Detection priority:** Check `package.json` → `go.mod` → `requirements.txt`/`pyproject.toml` → scan source imports → ask developer if ambiguous.

## Init: Entry Points to Scan

- Python: `app.py`, `main.py`, `__init__.py`, `wsgi.py`
- Node: `index.js`, `app.js`, `server.js`, `main.ts`
- Go: `main.go`, `cmd/*/main.go`

## Init: Parameter Extraction

- Path params: `/users/{id}`, `/users/:id`
- Query params: `request.args.get()`, `req.query`
- Body params: `request.get_json()`, `req.body`
- Types: from type hints, decorators, or validation

## Config File Format

**Always use folder includes** (`!include module/`), not individual file includes.

**Empty (init only):**
```yaml
# Generated by /generate-api-tests init
# Project: {{project-name}}
# Run tests: ./go-runner run --config spec-files/{{project-name}}-config.yaml

endpoints:
  # Tests will be added with /generate-api-tests create
  # - name: MODULE_NAME
  #   mode: default
  #   requests: !include module_name/
```

**With tests:**
```yaml
endpoints:
  - name: AUTH
    mode: default
    requests: !include auth/

  - name: USER
    mode: default
    requests: !include user/
```

````markdown
## Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `BASE_URL` | API base URL (required) | `http://localhost:5000` |

## Endpoints

### module_name

| Method | Path | Function | Description | Tested |
|--------|------|----------|-------------|--------|
| GET | /path | `function_name` | Description | No |
| POST | /path | `function_name` | Description | No |
````

**After generating tests, update `.api-spec.md`:**
1. Tested column: `No` → `Yes (YYYY-MM-DD)`
2. Coverage table: recalculate module and total counts
3. Coverage bar: update percentage and fill bar proportionally

**Coverage bar rendering:** Use `█` for filled and `░` for empty, 20 chars total. Formula: filled = round(percentage / 5). Examples:
- 0%: `░░░░░░░░░░░░░░░░░░░░` **0%**
- 25%: `█████░░░░░░░░░░░░░░░` **25%**
- 50%: `██████████░░░░░░░░░░` **50%**
- 75%: `███████████████░░░░░` **75%**
- 100%: `████████████████████` **100%**

## YAML Test Template

Each test file MUST start with a header documenting analyzed code paths:

```yaml
# Generated by /generate-api-tests create
# Endpoint: METHOD /path
# Source: module/file.py:line
#
# Code paths analyzed:
#   1. Happy path - primary flow (line XX)
#   2. Happy path - alternative flow (line YY)
#   3. ExceptionClass (controller.py:ZZ)
#   4. AuthorizationCheck (authorization.py:AA)

# === SETUP ===
- name: ENDPOINT_NAME - SETUP
  method: post
  path: /setup-endpoint
  body:
    name: "Test Fixture"
  vars:
    fixture_id: ${{ json.values.id }}$
  tests:
    - name: Setup successful
      assert: ${{ response.Status == 200 }}$

# === HAPPY PATH: Primary flow ===
- name: ENDPOINT_NAME - HAPPY PATH PRIMARY
  method: METHOD
  path: /endpoint-path
  body:
    required_param: ${{ env.REQUIRED_PARAM }}$
    resource_id: ${{ env.EXISTING_RESOURCE_ID }}$
  vars:
    result_id: ${{ json.values.id }}$
  tests:
    - name: Response status
      assert: ${{ response.Status == 200 }}$
    - name: Success status
      assert: ${{ json.status == 1 }}$
    - name: Has expected field
      assert: ${{ json.values.id != "" }}$

# === HAPPY PATH: Alternative flow ===
- name: ENDPOINT_NAME - HAPPY PATH WITH NEW SUB_RESOURCE
  method: METHOD
  path: /endpoint-path
  body:
    required_param: ${{ env.REQUIRED_PARAM }}$
    new_sub_resource:
      name: "Created via test"
  vars:
    result_id: ${{ json.values.id }}$
    sub_resource_id: ${{ json.values.sub_resource.id }}$
  tests:
    - name: Success status
      assert: ${{ json.status == 1 }}$
    - name: Sub-resource created
      assert: ${{ json.values.sub_resource != nil }}$

# === PARAM VALIDATION: Missing required_param ===
- name: ENDPOINT_NAME - MISSING REQUIRED_PARAM
  method: METHOD
  path: /endpoint-path
  body: {}
  tests:
    - name: Error status
      assert: ${{ json.status == 0 }}$

# === PARAM VALIDATION: Empty required_param ===
- name: ENDPOINT_NAME - EMPTY REQUIRED_PARAM
  method: METHOD
  path: /endpoint-path
  body:
    required_param: ""
  tests:
    - name: Error status
      assert: ${{ json.status == 0 }}$

# === PARAM VALIDATION: Invalid type ===
- name: ENDPOINT_NAME - INVALID TYPE NUMERIC_PARAM
  method: METHOD
  path: /endpoint-path
  body:
    required_param: ${{ env.REQUIRED_PARAM }}$
    numeric_param: "not_a_number"
  tests:
    - name: Error status
      assert: ${{ json.status == 0 }}$

# === BUSINESS RULE: Resource not found ===
- name: ENDPOINT_NAME - RESOURCE NOT FOUND
  method: METHOD
  path: /endpoint-path
  body:
    resource_id: "non_existent_id_000"
  tests:
    - name: Error status
      assert: ${{ json.status == 0 }}$

# === BUSINESS RULE: Quota exceeded ===
# NOTE: Needs specific setup to trigger quota limits
# - name: ENDPOINT_NAME - QUOTA EXCEEDED
#   ...

# === AUTH: No token ===
- name: ENDPOINT_NAME - UNAUTHORIZED
  method: METHOD
  path: /endpoint-path
  auth: none
  body:
    required_param: "value"
  tests:
    - name: Returns unauthorized
      assert: ${{ response.Status == 401 }}$

# === AUTH: Wrong owner ===
# NOTE: Requires second test user token in env (OTHER_USER_TOKEN)
# - name: ENDPOINT_NAME - FORBIDDEN WRONG OWNER
#   ...

# === CROSS-PARAM: Conflicting params ===
- name: ENDPOINT_NAME - CONFLICTING PARAMS
  method: METHOD
  path: /endpoint-path
  body:
    required_param: ${{ env.REQUIRED_PARAM }}$
    resource_id: ${{ env.EXISTING_RESOURCE_ID }}$
    new_sub_resource:
      name: "Should this be ignored?"
  tests:
    - name: Verify behavior
      assert: ${{ json.status == 1 || json.status == 0 }}$

# === TEARDOWN ===
- name: ENDPOINT_NAME - TEARDOWN
  method: delete
  path: /cleanup-path/${{ vars.result_id }}$
  tests:
    - name: Cleanup successful
      assert: ${{ response.Status == 200 }}$
```
